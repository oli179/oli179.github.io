【VM】windows server 2019 安裝 Docker 並架設 Gitlab

發表於 2023-07-28 分類於 Docker

說明

docker CPU至少給4核，記憶體要有8GB
上面這個規格是 docker 要的，所以實際需求會更大一些

需要連外網
windows server 2019 要先更新

下載 chrome
https://www.google.com/intl/zh-TW/chrome/

下載 VSCode(也可以用記事本)
https://code.visualstudio.com/download

首先先開啟以下功能

  1. Windows Hypervisor平台
  2. Windows 子系統 Linux 版
  3. 容器

如果是在windows可能名稱會有點不一樣，但是一看就知道是以上功能。

打開儀表板，選擇「 新增角色及功能 」

拉到最下面

安裝完成之後，要再回頭檢查看看是不是都有安裝完成。

如果是在實機安裝，Hyper-V有可能會安裝失敗。

如果失敗就要看一下 BIOS 中啟用虛擬化有沒有打開。

這篇文章可以參考一下

安裝完後，電腦要重啟

檢查有沒有 wsl

打開 PowerShell

wsl

如果出現上圖，表示有 wsl

安裝OpenSSH

確認 OpenSSH 可供使用

Get-WindowsCapability -Online | Where-Object Name -like 'OpenSSH*'

如果都沒有安裝會如下列：

Name  : OpenSSH.Client~~~~0.0.1.0
State : NotPresent

Name  : OpenSSH.Server~~~~0.0.1.0
State : NotPresent

一般來說都會是 Server 沒有安裝

安裝完之後要去工作管理員看是不是執行中
如果不是就按右鍵啟動
啟動完後要稍微等一下
以免啟動失敗沒有看到

安裝 OpenSSH Server or Client

看哪個沒裝就裝哪個

# Install the OpenSSH Client
Add-WindowsCapability -Online -Name OpenSSH.Client~~~~0.0.1.0

# Install the OpenSSH Server
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0

Download Docker Desktop

在安裝的時候，電腦必須是更新過的。

安裝需要一點時間。

安裝完後重新開機（會自動重新開機）
有時候不會重開機

重開機之後，Docker會自動啟動，如果 windows 沒有更新，可能會出現警告訊息。
用了幾台虛擬機測試，如果不更新也不會有什麼問題，但是如果未來要更新 Docker 可能會無法更新。

接下來會請你登入

不登入也可以

進入 Docker 畫面

設定環境資源

虛擬機測試安裝，建議 CPU 至少 4，記憶體至少 8GB ，Swap 2GB，GitLab 跑起來比較不會卡。

Git downloads

安裝測試一下有沒有安裝成功：

git --version

下載 GitLab

在 Docker 最上方的搜尋列內輸入 「 gitlab-ce 」

選擇下載數較高的，然後按 「 Pull 」，下載會需要一點時間。

這個時候 Images 就下載了 gitlab

設定GitLab資料儲存的地方（有三種方法）

方法一：docker-compose

新增以下資料夾：(自訂)

C:\Docker_Data\GitLab03_Data\config

新增 docker-compose.yml

檔案通常我會建在Docker_Data的同層

檔案內容：

version: '3'
services:
  gitlab:
    container_name: gitlab03
    image: gitlab/gitlab-ce:latest
    restart: always
    ports:
      - 8080:80
      - 4433:443
      - 2222:22
    volumes:
      - 'E:\Docker_Data\Gitlab Server\config:/etc/gitlab'
      - 'E:\Docker_Data\Gitlab Server\logs:/var/log/gitlab'
      - 'E:\Docker_Data\Gitlab Server\data:/var/opt/gitlab'

以上面寫法來說～
volumes的設定會把容器的資料放在E槽裡面，放在E槽裡可以方便查看內容，如果不小心把容器刪除掉，也不用擔心資料會不見。可是目前這種寫法，雖然可以寫入資料，但是 Gitlab 會無法啟動成功，原因還沒有去查。曾經有一次使用舊版且別人優化過的 GitLab Image，有成功啟動過，不過因為太舊，所以就沒有使用了。

後來改用下列的寫法～

另一個寫法：

version: '3'
services:
  gitlab:
    container_name: gitlab01
    image: gitlab/gitlab-ce:latest
    restart: always
    ports:
      - 8080:80
      - 4433:443
      - 2222:22
    volumes:
      - E:\docker\gitlab\config:/etc/gitlab
      - gitlab-logs:/var/log/gitlab
      - gitlab-data:/var/opt/gitlab
volumes:
  gitlab-logs:
  gitlab-data:

container_name: 是設定容器的名稱。

volumes: 是 gitlab01 容器資料要放置的地方。

config：後面是原本容器資料的擺放位置，這個寫法是要將後者映射到前者。

volumes:
      - E:\docker\gitlab\config:/etc/gitlab
      - gitlab-logs:/var/log/gitlab
      - gitlab-data:/var/opt/gitlab
      
# E槽的資料就儲存在上列所設的位址
# 而 gitlab-logs 跟 gitlab-data 則是儲存在 Docker 裡面

啟動 images

打開 PowerShell

將位置移到 docker-compose.yml 的同一層。

然後輸入指令：

docker-compose -p docker-compose up -d

指令的第二個 docker-compose 是檔案名稱

這個指令也會順便啟動 Images

第一次啟動可能會久一些

啟動成功

第二種方法 - 直接用 docker run 指令

docker run -d --hostname localhost --publish 4433:443 --publish 8080:80 --publish 2222:22 --name gitlab --restart always --volume “C:/Docker_Data/GitLab_Data/config”:/etc/gitlab --volume “C:/Docker_Data/GitLab_Data/logs”:/var/log/gitlab --volume “C:/Docker_Data/GitLab_Data/data”:/var/opt/gitlab --user win-806blb7krml --shm-size 256m gitlab/gitlab-ce:latest

很多人都使用這種方法，但是我一直都試不成功，雖然指令下了之後，映射資料夾都會自動設定好，裡面也會有資料寫進去，但是 GitLab 網頁就一直沒有辦法打開，所以這個方法我是不推薦的。

第三種方法 - 在啟動時直接設定

第一次啟動時，按這個啟動鍵，會出現一個設定畫面，可以設定端口及儲存的位址，這個方法只要設定資料儲存，GitLab 都無法啟動成功，如果只設定端口，GitLab 就可以正常啟動，在網路上看了很多資料，大部份都說是權限的問題，但是還沒有成功處理好，所以我也不推薦這個方法。

gitlab 初次密碼

密碼要在安裝後24小時內用指令取得，不然就拿不出來了。

docker exec -it gitlab2 grep 'Password:' /etc/gitlab/initial_root_password

成功進入

進入後馬上更換密碼

更換密碼後會強制重新登入

新增使用者

在新增使用者後，要再進入使用者的頁面新增密碼，這個密碼是臨時密碼，當使用者用臨時密碼登入時，系統會要求使用者更新密碼。

更新後記得要按儲存

使用者要進入 GitLab 時，可以輸入 Username 或是 Email 都可以。

設定 ssh key

查看是否有 ssh

ls -al ~/.ssh

建 SSH Key

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

下指令顯示SSH Key

cat ~/.ssh/id_rsa.pub

把生產出來的文字都拷貝起來

將公鑰放到 GitLab

找到個人頭像，然後選擇 Preferences

找到 SSH Keys

把剛剛拷貝起來的文字貼到 Key 的框框內

然後 Title 自取，填好就按最下面的 Add Key

測試是否可以新增專案

新增專案

以下表示新增成功

查看 clone 位址的主機名稱是不是一串亂英數值

就像這樣：

git@30b7e9bdda7c:root/0705Project.git

30b7e9bdda7c 電腦無法解析，所以要更改。

更改的方式請看下列連結：

連結：GitLab容器上的項目進行clone時，IP主機位址顯示一串數字而不是正常IP地址

測試 git clone ssh

  • 如果沒有放公鑰到 GitLab 裡面，就無法使用 ssh 來下載

測試 git clone http

  • 如果無法 git clone http 多半是端口沒開
  • 因為是 VM ，所以在主機後面要加 8080 （8080是我設的）
  • 遠端伺服器管理工具要開
  • 網頁伺服器要打開
