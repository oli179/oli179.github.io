公證的基本介紹

什麼是公證 (Notarization)：

  • 公證是 Apple 提供的安全檢查服務，用於驗證應用是否安全且未被惡意修改。
  • 從 macOS Catalina (10.15) 開始，公證已成為非 App Store 分發應用的必要步驟。

為什麼需要公證：

  • 確保應用可以正常通過 Gatekeeper。
  • 增強用戶對應用的信任度，避免被標記為「無法驗證的開發者」。

公證的流程

準備開發者帳號：

  • 有效的 Developer ID（付費會員）。

對APP進行簽名：

  • 使用 codesign 為APP及其”所有執行檔”進行簽名。

提交公證請求：

  • 使用 xcrun notarytool 將簽名後的應用提交到 Apple 的公證服務。

公證的注意事項

支援的檔案格式：
.app：獨立應用程序。
.dmg：安裝映像文件。
.pkg：安裝包。
.zip：壓縮應用。

需要簽名的檔案：

  • 主應用程序及其包含的所有二進制檔案（如 .sh、.dylib、內嵌的 .app 等）。
  • 含有未簽名的檔案會導致公證失敗（很重要）。

有效的憑證：

  • 確保使用的 Developer ID 憑證有效且未過期。
  • 必須使用 “相同” 的帳號進行簽名和公證（這個很重要）。

App Store 與非 App Store 的區別：

  • App Store 分發應用會自動進行公證，開發者無需額外操作。
  • 非 App Store 分發需要手動完成公證（就是現在要做的）。

官方文件：Apple官方公證指南

公證實際操作

notarize-profile設置

notarize-profile 主要用來簡化公證工具的配置過程。
請參考Apple官網
https://developer.apple.com/documentation/technotes/tn3147-migrating-to-the-latest-notarization-tool

notarize-profile 的作用

簡化公證流程，減少重複輸入的參數。
自動填充公證請求中必要的資訊：

  • Apple ID
  • App-specific password

使用 notarize-profile 可以避免每次執行公證指令時重複提供這些細節。

上傳公證（事後查詢公證狀態）

xcrun notarytool submit /path/to/xxx.dmg --keychain-profile notarize-profile

上傳公證（現場等）

xcrun notarytool submit /path/to/xxx.dmg --keychain-profile notarize-profile --wait

這個寫法會直接返回結果，不會立即退出。

查詢公證結果

取消公證

取消公證請求的操作僅適用於已提交但尚未完成的公證。

xcrun altool --notarization-info "REQUEST-UUID" \
  --username "your-apple-id@example.com" \
  --password "app-specific-password"

下載公證票據並附加到文件中

xcrun stapler staple /path/to/file

驗證票據是否附加成功

xcrun stapler validate /path/to/file

如果完成了 .dmg 或應用的 公證（notarization），但 未附加票據（staple），可能會出現以下問題：

用戶在離線環境中無法驗證應用

  • 未附加票據：
    當用戶嘗試打開應用或安裝 .dmg 時，macOS 需要連接到 Apple 的公證服務來驗證應用的合法性。如果用戶處於 離線環境，系統將無法驗證，可能會顯示安全警告，例如：”應用程式無法打開，因為無法驗證其來源。”
  • 附加票據後：
    公證票據會被嵌入到應用或 .dmg 文件中，用戶在 離線 時仍然可以正常打開或安裝，因為系統可以直接從文件中讀取票據。
