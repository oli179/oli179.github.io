Wallet錢包測試-一鍵加入錢包

發表於 2024-12-17 分類於 Wallet
生成 .pkpass 測試文件(創建文件、打包和簽名過程)

🌱環境：
MacOS: Sonoma 14.3（23D56）
Xcode: Version 15.4 (15F31d)

1. 必要條件

  • Apple 開發者付費帳戶（個人或企業都可以）：
    • AppIDs:要新增Wallet功能。
    • Certificates:要申請Pass Type ID憑證。
  • XCode開發工具：
    • 在 Signing & Capabilities要設定Wallet

2. 檔案結構

.pkpass 是一個壓縮包（.zip），其中包含以下文件：

2.1 必須文件

1. pass.json：核心配置文件，包含 Pass 的基本屬性。
2. manifest.json：所有文件的哈希值清單(SHA1 hash )。
3. signature：使用 Apple 開發者證書簽署的 manifest.json 文件。

2.2 可選資源

  • 圖片資源：
    • icon.png 和 icon@2x.png（必須有，用於顯示在 Wallet 中的圖標）。
    • 其他可選圖片（如 logo.png、background.png）。
    • 圖片大小有其限制。

Wallet圖片規範請參考Wallet開發-Wallet圖片限制

生成步驟

💦 步驟 1：準備 pass.json 文件

創建一個名為 pass.json 的文件，並填寫以下內容（這是一個範例）：

{
  "formatVersion": 1,
  "passTypeIdentifier": "pass.com.yourcompany.test",
  "serialNumber": "123456789",
  "teamIdentifier": "ABCDE12345",
  "organizationName": "Your Company",
  "description": "Test Pass",
  "logoText": "Wallet Test",
  "foregroundColor": "rgb(255,255,255)",
  "backgroundColor": "rgb(0,0,255)",
  "barcode": {
    "format": "PKBarcodeFormatQR",
    "message": "https://www.example.com",
    "messageEncoding": "iso-8859-1"
  }
}

💦 步驟 2：準備圖片文件

  • 必要圖片：
    • icon.png：大小 180x180 px。
    • icon@2x.png：大小 360x360 px（高分辨率版本）。
  • 可選圖片：
    • logo.png：大小自定義，用於 Pass 的主界面。

💦 步驟 3：生成 manifest.json 文件

使用 SHA1 計算每個文件的哈希值（包括 pass.json 和所有圖片文件）。

可以用指令產生

manifest.json，範例如下：

{
  "pass.json": "3fc95fdf2f1e9968dfb40bf61ae65e55e76261b8ec764015b235cc7dd6b855a5",
  "icon.png": "f2c65fdf78ae2aa72e685d73d3ebead45f82a5a1f6637978be8f6e8318f6c1e0",
  "icon@2x.png": "a1f0ac9efde9e215b47553fd51a156f8b5e899e9b4c71a1a4c9552c7fade9482"
}

💦 步驟 4：簽名 manifest.json 文件

使用Pass Type ID 憑證及私鑰簽名：
憑證下載匯入後（.cer）要將憑證與私鑰匯出為.p12然後轉成.pem。
做法可以參考 Pass Type ID憑證的處理步驟 。

使用指令執行簽名 manifest.json，生成 signature 文件：

openssl smime -binary -sign -signer certificate.pem -inkey privateKey.pem -in manifest.json -out signature -outform DER

certificate.pem：你的 Apple Pass Type ID 憑證（可從 Apple 開發者網站下載）。
privateKey.pem：你的私鑰文件。

💦 步驟 5：生成 .pkpass 文件

將要打包的資源全部選取後，按右鍵選壓縮，預設名稱為 “封存.zip”。
打包成壓縮文件並更名為 .pkpass（例如 TestPass.pkpass）。

這個文件就是要用來測試Wallet電子錢包用的。

SWIFT測試程式


import UIKit
import PassKit

class ViewController: UIViewController {

    override func viewDidLoad() {
        super.viewDidLoad()
        setupUI()
    }

    func setupUI() {
        let addPassButton = UIButton(type: .system)
        addPassButton.setTitle("新增票券", for: .normal)
        addPassButton.addTarget(self, action: #selector(addPass), for: .touchUpInside)
        addPassButton.frame = CGRect(x: 50, y: 100, width: 200, height: 50)
        view.addSubview(addPassButton)
    }

    @objc func addPass() {
        if let passURL = Bundle.main.url(forResource: "TestPass", withExtension: "pkpass") {
            print("票券檔案路徑：\(passURL)")
            let fileExists = FileManager.default.fileExists(atPath: passURL.path)
            print("檔案存在：\(fileExists)")
        } else {
            print("無法取得票券檔案的 URL")
        }
        // 加入票券的邏輯
        guard let passURL = Bundle.main.url(forResource: "TestPass", withExtension: "pkpass"),
              let passData = try? Data(contentsOf: passURL),
              let pass = try? PKPass(data: passData) else {
            print("無法讀取票券檔案")
            return
        }

        let passVC = PKAddPassesViewController(pass: pass)
        present(passVC!, animated: true, completion: nil)
    }
}
