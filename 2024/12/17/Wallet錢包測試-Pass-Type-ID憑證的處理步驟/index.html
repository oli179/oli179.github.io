Wallet錢包測試-Pass Type ID憑證的處理步驟

安裝 pass.cer

雙擊下載的 pass.cer 文件，將其安裝到 macOS 的 Keychain Access（鑰匙圈）。
可以在 “我的憑證” 或 “系統” 鑰匙圈中看到該憑證。

❌ 在這裡遇到問題，一開始下載的pass.cer不含私鑰，後來安裝了WWDR-G4憑證就成功了。
在Certificates, Identifiers & Profiles新增憑證時，下方會有可以下載WWDR-G4的連結。

匯出 .p12 檔案

為了在命令行工具或其他應用中使用憑證，需要將憑證和私鑰匯出為 .p12 文件，然後再轉成.pem。

❌ 在驗證憑證時發生了錯誤：

其中 error 2…意指缺少必要的中間 CA 憑證，導致 Apple 的 WWDR 憑證無法驗證，需要下載 Apple Root CA - G3 憑證（.pem 格式），載點：https://www.apple.com/certificateauthority/

將憑證匯出為.pem

網路上有可以用指令可以將.p12轉成.pem，但是試了幾個，都無法成功轉成.pem，最後試了一個，但是有需要手動存檔。（機器人的方式也是跟網路一樣…不行）。

💦 步驟1. 先在keyChain將Pass Type ID憑證匯出為pass.p12，然後用指令執行：

openssl pkcs12 -in pass.p12 -legacy -nodes

如果成功會出現一大串的文字，裡面包含了憑證跟私鑰。

💦 步驟2. 分別單獨保存憑證與私密金鑰

用文字編輯器分別保存：

  • 保存憑證：
    將 —–BEGIN CERTIFICATE—– 到 —–END CERTIFICATE—– 的內容複製到 certificate.pem 文件中。
  • 保存私密金鑰：
    將 —–BEGIN PRIVATE KEY—– 到 —–END PRIVATE KEY—– 的內容複製到 privateKey.pem 文件中。

💦 步驟3. 測試 .pem 文件是否有效

用指令驗證私密金鑰是否有效：

openssl rsa -in privateKey.pem -check

驗證憑證是否有效：

openssl x509 -in certificate.pem -text -noout

驗證私密金鑰與憑證是否匹配：

openssl x509 -noout -modulus -in certificate.pem | openssl md5
openssl rsa -noout -modulus -in privateKey.pem | openssl md5

兩個命令的輸出結果應相同，表示私密金鑰與憑證是匹配的。
