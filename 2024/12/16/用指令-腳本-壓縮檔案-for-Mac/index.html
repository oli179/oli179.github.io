用指令(腳本)壓縮檔案 for Mac

發表於 2024-12-16 分類於 工具

三種方式：

  1. 無密碼
  2. 有密碼
  3. 有密碼（亂數密碼）

1.無密碼

終端機指令：

zip -r SecuComp.zip 檔案

-r 壓縮的意思
SecuComp.zip 是壓縮後的名稱
檔案 是要壓縮的檔案名稱

2.有密碼

終端機指令：

zip -er SecuComp.zip /Users/Desktop/已完成公證

e 加上密碼
-r 壓縮的意思
SecuComp.zip 是壓縮後的名稱
已完成公證 是要壓縮的檔案名稱

指令輸入並按下enter之後會要求輸入二次密碼

3.有密碼（亂數密碼）

這種方式要用寫腳本的方式進行

腳本

#!/bin/bash

# 檢查是否提供輸入檔案路徑參數
if [ -z "$1" ]; then
  echo "Usage: $0 <file_or_directory_to_compress> [output_zip_path]"
  exit 1
fi

# 取得輸入的檔案或目錄路徑
input_path=$1

# 檢查檔案或資料夾是否存在
if [ ! -e "$input_path" ]; then
  echo "Error: File or directory '$input_path' does not exist."
  exit 1
fi

# 取得輸出的壓縮檔案路徑，若未提供，使用預設名稱 output.zip
output_path=${2:-output.zip}

# 生成隨機密碼
password=$(openssl rand -base64 6)
echo "Generated password: $password"

# 進入輸入路徑的父目錄
cd "$(dirname "$input_path")"

# 壓縮並加密目錄或檔案（僅包含目標的名稱而非完整路徑）
zip -P "$password" -r "$output_path" "$(basename "$input_path")"

# 提示壓縮結果
if [ $? -eq 0 ]; then
  echo "Successfully compressed '$input_path' into '$output_path'"
  echo "Password: $password"
else
  echo "Error: Failed to compress the file or directory."
fi

重點：

  1. 亂數密碼為六碼
  2. 如果沒有提供輸出名稱，預設為output.zip

給權限

chmod +x compress_with_password.sh

如果執行結果為找不到compress_with_password.sh就改寫完整的路徑

執行壓縮作業

//.sh檔案放哪裡，就在哪裡執行
cd /Users/Downloads

./compress_with_password.sh /Users/olit/Desktop/xxxx /Users/zzz/abc.zip
# Mac
